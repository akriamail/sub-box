# AGENTS.md — sub-box 独立仓库指引

> 与 `CLAUDE.md` 保持绝对同步。任何一方更新时，另一方必须同步。

## 仓库边界

`services/sub-box/` 是一个独立 GitHub 仓库，目录内自带 `.git/`。

操作时必须把本目录视为独立仓库，而不是父仓库 `insightful-ops` 的普通子目录。

## 强制规则

1. 禁止删除、移动、重写或清理本目录内的 `.git/`，除非用户明确要求。
2. 禁止为了纳入父仓库而自动把本目录改成普通目录、submodule 或 subtree。
3. 执行 Git 命令时，必须明确当前目标仓库：
   - 操作 sub-box：使用 `git -C services/sub-box ...`
   - 操作父仓库：在父仓库根目录执行，并注意不要误处理 sub-box 内部仓库。
4. 任何 `git commit` / `git tag` / `git push` 都属于高风险操作，必须先给计划并等待用户回复 `批准执行`。
5. 本目录内的源码、脚本、文档修改属于本地 B 类修改，可按任务直接编辑，但完成后必须说明改动范围。

## 目录结构约定

- 根目录只放用户入口和项目说明：`manager.sh`、`README.md`、`AGENTS.md`、`CLAUDE.md`。
- 可执行运行脚本放入 `bin/`：`sub-box.sh`、`update.sh`、`fetch_ext.sh`。
- 功能模块放入 `lib/`。
- 不保留废弃入口脚本，确认无用后应删除，避免误导部署。
- 不要把新脚本继续平铺到根目录；新增脚本优先放到 `bin/` 或 `lib/`。

## 维护要求

- 修改本文件时，必须同步修改同目录的 `CLAUDE.md`，内容保持一致。
- 如果发现父仓库与本独立仓库的 Git 状态混淆，先停下来说明，不要自行转换仓库结构。
